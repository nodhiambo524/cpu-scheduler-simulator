# cpu-scheduler-simulator

DESCRIPTION
A software simulation that demonstrates how Operating Systems decide which processes get CPU time and how different scheduling algorithms affect performance. Different scheduling algorithms can significantly impact system performance, resource utilization, and user experience. This project explores these tradeoffs by implementing and comparing common CPU scheduling algorithms. Through this simulator, users can input processes, run scheduling algorithms, and analyze performance metrics such as waiting time and turnaround time, and compare averages of both waiting time and turnaround time.

OVERVIEW
This project explores software and a systems engineering perspective to show how programming and performance analysis can be used in decision-making. It also serves as an opportunity to deepen understanding of scheduling, optimization, and systems design. 

PROJECT GOALS
- Simulate common CPU scheduling algorithms
- Compare algorithm performance
- Calculate scheduling metrics
- Visualize scheduling decisions and tradeoffs
- Strengthen understanding of OS concepts
